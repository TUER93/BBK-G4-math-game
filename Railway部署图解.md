## 🎯 关键配置项

### ✅ 必须配置

1. **生成域名**
   - 位置：Settings → Networking → Generate Domain
   - 作用：获得访问网址

2. **添加 Volume**
   - 位置：Settings → Volumes → New Volume
   - Mount Path: `/app`
   - 作用：数据持久化，防止数据丢失

### ⚙️ 可选配置

3. **环境变量**（通常不需要）
   - `NODE_ENV=production`
   - `PORT` 会自动配置

## ❓ 常见问题图解

### Q1：找不到我的仓库怎么办？

```
Railway → New Project
    ↓
Deploy from GitHub repo
    ↓
看不到仓库？
    ↓
点击 [Configure GitHub App]
    ↓
选择 [All repositories] 或指定仓库
    ↓
保存返回
    ↓
重新选择仓库
```

### Q2：部署失败怎么办？

```
项目页面 → Deployments 标签页
    ↓
点击失败的部署记录
    ↓
查看 Logs（日志）
    ↓
找到错误信息（红色文字）
    ↓
根据错误修复代码
    ↓
git push 重新部署
```

### Q3：如何查看运行日志？

```
项目页面 → Logs 标签页
    ↓
实时查看服务器输出
    ↓
看到 "服务器已启动" 表示正常
```
